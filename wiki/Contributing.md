# Contributing

Thank you for contributing! This project welcomes issues and pull requests.

How to contribute
1. Fork the repository
2. Create a branch: `git checkout -b feature/my-change`
3. Make changes and include tests where applicable
4. Run linting and tests (if present)
5. Open a pull request describing your changes

Coding guidelines
- Keep changes focused and minimal
- Follow existing code style and naming conventions
- Avoid changing unrelated files

Testing
- Unit tests are welcomed; add tests under `test/` if present
- For manual testing, run server locally and exercise UI pages

Documentation
- Add or update wiki pages if behaviour or setup changes
- Update `CHANGELOG.md` for notable changes

Security
- Do not commit secrets or credentials
- Use `config.js` env overrides for sensitive data

Contact
- Open issues or PRs on GitHub; tag maintainers when needed
